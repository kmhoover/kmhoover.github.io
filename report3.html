My Main Tasks:

eSentire sells a sensor that continuously monitors clients' networks for threats. The sensor reports all information to the Security Operations Centre and blocks traffic that violates a rule. As a Quality Assurance Developer, I was responsible for testing this sensor. I developed a test suite from scratch that replicated each event triggering an alert and verified that the correct rule generated the alert.

Moreover, I collaborated with the Sensor team to identify and report any issues or bugs found during the testing process. This co-op placement gave me a better understanding of the importance of quality assurance in the cybersecurity industry and the challenges involved in developing and testing complex systems. Additionally, I gained valuable connections in the industry and hands-on experience with cutting-edge technologies that will benefit me in my future career.

Test Automation:

One of the primary obstacles I encountered was automating the test suite. To accomplish this, I needed to log in to the sensor, execute a script that breached one of the security rules, and then validate that the proper alert was generated without requiring any user intervention. I found this to be an exciting challenge, as it provided me with a deeper understanding of shell scripting and allowed me to automate commands using a Python script.

Sensor Rules:

I tested both Intrusion Detection Systems (IDS) and Intrusion Prevention Systems (IPS). To carry out these tests, I had to evaluate the rules that the sensor used to either report suspicious network traffic or block malicious network traffic. To create automated tests for these rules, I needed a comprehensive understanding of how the sensor operated and the specific rules that were in place. Through this experience, I gained valuable expertise in network security and the intricacies involved in testing and validating IDS/IPS systems.
